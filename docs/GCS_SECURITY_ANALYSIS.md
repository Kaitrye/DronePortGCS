# Анализ безопасности системы управления беспилотниками (НУС)

## Пункт 1. Ключевые активы, оценка уровня ущерба и приемлемость риска

### 1.1 Идентификация активов

Активы - это данные, сервисы и инфраструктура системы управления дронами (НУС), которые имеют ценность и должны быть защищены.

| №  | Актив                        | Описание                                                 | Категория      |
| -- | ---------------------------- | -------------------------------------------------------- | -------------- |
| N1 | Миссии дронов                | План полёта: координаты маршрута, точки взлёта и посадки. | Данные         |
| N2 | Телеметрия дронов            | Координаты, скорость, заряд батареи, состояние дрона.     | Данные         |
| N3 | Журнал событий системы       |Логины, пароли, токены и ключи доступа сервисов системы.   | Данные         |
| N4 | Учётные данные системы    | Логины, пароли и токены доступа операторов.               | Секреты        |
| N5 | Сервис НУС                  | Актив системы, включающий координацию миссий, управление дронами, хранение телеметрии и статусов дронов, а также обработку команд эксплуатанта.  | Сервис         |

### 1.2 Оценка уровня ущерба

Оценка проводится по трём свойствам информационной безопасности (**CIA**):

* **Confidentiality** - конфиденциальность
* **Integrity** - целостность
* **Availability** - доступность

Шкала оценки:

| Уровень     | Описание                       |
| ----------- | ------------------------------ |
| Низкий      | Незначительные последствия для системы. Нарушение влияет только на отдельные данные или сервисы и может быть быстро восстановлено без заметного влияния на выполнение миссий.   |
| Средний     | Заметные последствия для работы системы. Возможна временная потеря части функциональности или необходимость ручного восстановления данных.          |
| Высокий     | Серьёзные последствия для системы управления дронами. Может привести к нарушению выполнения миссий или деградации работы сервисов.          |
| Критический | Катастрофические последствия для системы. Возможна потеря управления дронами, нарушение безопасности полётов или остановка работы ключевых компонентов системы. |

## Пункт 5. Моделирование угроз

### Угроза 1 - Подмена команд миссий

**Описание:**
Злоумышленник отправляет команду управления дроном от имени НУС или эксплуатанта. В результате дрон может выполнить неверные действия или начать выполнение ложной миссии.

**Нарушаемые цели:**

ЦБ-1 (использование только подлинных миссий)
ЦБ-3 (выполнение только аутентичных и авторизованных команд)

**Критичность:**
**КРИТИЧЕСКАЯ 🔴**

**Контрмеры:**

* Аутентификация сервисов и эксплуатанта (сертификаты / ключи)
* Цифровая подпись команд миссий
* Проверка авторизации отправителя
* Логирование всех команд с метаданными (timestamp, ID миссии)

---

### Угроза 2 - Компрометация секретов

**Описание:**
Злоумышленник получает доступ к ключам, токенам или учётным данным системы и может использовать их для несанкционированного доступа к сервисам НУС.

**Нарушаемые цели:**

ЦБ-2 (конфиденциальность информации)
ЦБ-3 (выполнение только авторизованных команд)

**Критичность:**
**КРИТИЧЕСКАЯ 🔴**

**Контрмеры:**

* Хранение секретов в защищённом хранилище (Vault, Docker Secrets)
* Ограничение доступа к секретам только авторизованным сервисам
* Регулярная ротация ключей и токенов
* Аудит доступа к секретам

---

### Угроза 3 - Отказ в работе сервисов (DoS)

**Описание:**
Злоумышленник отправляет большое количество запросов к серверу Нуса, что приводит к перегрузке сервисов и невозможности обработки миссий и телеметрии.

**Нарушаемые цели:**

ЦБ-3 (невозможность обработки авторизованных команд)
ЦБ-4 (нарушение передачи критичной информации)

**Критичность:**
**ВЫСОКАЯ**

**Контрмеры:**

* Rate limiting для внешних и внутренних сервисов
* Балансировка нагрузки
* Мониторинг состояния сервисов
* Использование очередей и Store для буферизации сообщений

## Пункт 6. Домен доверия
Таблица отражает распределение компонентов **НУСа** по уровням доверия.
 Компоненты в группе **Trusted** критичны для целостности и доступности системы, **Semi-trusted** взаимодействуют с внешними источниками и требуют проверки данных, а **Untrusted** находятся вне контроля системы и могут быть потенциально вредоносными.

| Уровень доверия | Компоненты                                           | Обоснование |
| --------------- |------------------------------------------------------| ----------- |
| Untrusted       | Эксплуатант, дроны.                                  | Внешние субъекты, не находятся под контролем системы. Команды и телеметрия могут быть поддельными или вредоносными. |
| Semi-trusted    | Orchestrator, MissionStore, DroneStore, DroneManager. | Взаимодействуют с внешними компонентами. Компрометация может повлиять на маршрутизацию команд и управление ресурсами. |
| Trusted         | MissionConverter, PathPlanner. | Критические внутренние сервисы НУС. Обеспечивают целостность миссий и телеметрии, управление дроном и хранение состояния системы. |
