# Анализ безопасности Дронопорта

## Пункт 1. Ключевые активы, оценка уровня ущерба и приемлемость риска

### 1.1 Идентификация активов

Активы - это ресурсы системы Дронопорта, которые используются для обеспечения посадки, обслуживания и зарядки дронов и требуют защиты с точки зрения информационной безопасности.

| №  | Актив                        | Описание                                                                               | Категория      |
| -- | ---------------------------- |----------------------------------------------------------------------------------------| -------------- |
| D1 | Данные дронопорта            | Состояние посадочных площадок, зарядных станций.                                       | Данные         |
| D2 | Данные дронов            | Данные о статусе дрона при посадке и обслуживании.                                     | Данные         |

| D5 | Учётные данные системы    | Логины, пароли и токены доступа операторов.                                             | Секреты        |

### 1.2 Оценка уровня ущерба

Оценка проводится по трём свойствам информационной безопасности (**CIA**):

* **Confidentiality** - конфиденциальность
* **Integrity** - целостность
* **Availability** - доступность

Шкала оценки:

| Уровень     | Описание                       |
| ----------- | ------------------------------ |
| Низкий      | Незначительные последствия для системы. Нарушение влияет только на отдельные данные или сервисы и может быть быстро восстановлено без заметного влияния на выполнение миссий.    |
| Средний     | Заметные последствия для работы системы. Возможна временная потеря части функциональности или необходимость ручного восстановления данных.           |
| Высокий     | Серьёзные последствия для инфраструктуры дронопорта. Может привести к нарушению выполнения миссий или деградации работы сервисов.          |
| Критический | Катастрофические последствия для системы. Возможна потеря управления дронами, нарушение безопасности полётов или остановка работы ключевых компонентов системы. |

## Пункт 5. Моделирование угроз


### Угроза 1 - Компрометация секретов

**Описание:**
Злоумышленник получает доступ к ключам, сертификатам или другим секретным данным системы. Это может позволить ему выдавать себя за доверенный сервис или дрон и отправлять ложные команды или данные.

**Нарушаемые цели:**

ЦБ-2 (выполнение только аутентичных и авторизованных команд)
ЦБ-3 (передача подлинной информации)

**Критичность:**
**КРИТИЧЕСКАЯ 🔴**

**Контрмеры:**

* Использование защищённого хранилища секретов
* Ограничение доступа к ключам и сертификатам
* Регулярная ротация ключей
* Аудит доступа к секретам

---

### Угроза 2 - Отказ в обслуживании (DoS)

**Описание:**
Злоумышленник отправляет большое количество сообщений или запросов в сервисы Дронпорта. Это может привести к перегрузке системы и невозможности обработки команд управления и телеметрии.

**Нарушаемые цели:**

ЦБ-2 (невозможность обработки авторизованных команд)
ЦБ-3 (нарушение передачи критичной информации)

**Критичность:**
**ВЫСОКАЯ**

**Контрмеры:**

* Rate limiting
* Балансировка нагрузки
* Использование очередей сообщений
* Мониторинг состояния и нагрузки сервисов

---

### Угроза 3 - Несанкционированный доступ к сервисам DronePort

**Описание:**
Злоумышленник пытается получить доступ к внутренним сервисам Дронпорта (например, Orchestrator или PortManager) без авторизации. В случае успешной атаки он может отправлять команды управления инфраструктурой, изменять состояние портов или получать доступ к данным системы.

**Нарушаемые цели:**

ЦБ-1 (нарушение корректного использования посадочных портов)
ЦБ-2 (выполнение неавторизованных команд)
ЦБ-3 (нарушение целостности данных системы)

**Критичность:**
**КРИТИЧЕСКАЯ 🔴**

**Контрмеры:**

* Аутентификация и авторизация сервисов
* Использование сертификатов или ключей для межсервисного взаимодействия
* Ограничение доступа к внутренним API
* Логирование и аудит всех запросов к сервисам

## Пункт 6. Домен доверия

 Таблица отражает распределение компонентов **Дронопорта** по уровням доверия.
 Компоненты в группе **Trusted** критичны для целостности и доступности системы, **Semi-trusted** взаимодействуют с внешними источниками и требуют проверки данных, а **Untrusted** находятся вне контроля системы и могут быть потенциально вредоносными.

| Уровень доверия | Компоненты                                                        | Обоснование                                                                   |
| --------------- |-------------------------------------------------------------------| ----------------------------------------------------------------------------- |
| Untrusted       | Дроны, SITL, эксплуатант, ChargingManager.                                        | Внешние субъекты, команды и телеметрия могут быть поддельными                 |
| Semi-trusted    | DronePortOrchestrator,   StateStore, DroneManager,  DroneRegistry. |                             Взаимодействуют с внешними компонентами, требуется проверка данных            |
| Trusted         | PortManager.                                                       | Критические внутренние сервисы, обеспечивают целостность и доступность данных |
